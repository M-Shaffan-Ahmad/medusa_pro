The LLM Inference Challenge

  • Memory Bottleneck: Requires loading entire model weights on memory
  • Sequential Generation: Tokens generated one-by-one, inherently slow
  • Computation-Memory Mismatch: Each token requires fetching model parameters
  • Latency Critical: Real-time applications suffer from slow token generation
Goal: Accelerate token generation by predicting multiple tokens simultaneously while maintaining quality

Medusa: Multi-Head Decoding Architecture

  • Key Innovation: Add extra "prediction heads" to LLM for speculative decoding
  • Architecture: Multiple decoding heads predict next K tokens in parallel
  • Tree-Based Attention: Process multiple candidates efficiently without context mixing
  • Speedup: 2.2x - 3.6x faster generation compared to baseline
Key Advantage: Original model remains unchanged - only new heads are fine-tuned!

TurboQuant: Efficient Quantization

  • What: Reduces model size by converting weights to lower precision (int8/fp8)
  • Why: Reduces memory footprint, faster computation, enables deployment on resource-constrained devices
  • Integration: Apply quantization to base model + prediction heads
  • Trade-off: Minimal quality loss while achieving 4x memory reduction
Medusa + TurboQuant: 2-3x speedup + 4x memory reduction = optimal efficiency!

Training Medusa Heads

Medusa-1: Head-Only Training

  • Freeze base model weights
  • Train only new prediction heads
  • Requires less data and computation
  • Suitable for GPU-poor scenarios

Medusa-2: Full Model Training

  • Joint training of base model + heads
  • Better quality but more computationally expensive
  • Requires special recipe to maintain original model performance
